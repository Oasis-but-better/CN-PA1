About This Project

Simple HTTP Server Implementation - PA1

Project Overview

This project implements a fully functional HTTP/1.1 server from scratch using Python and raw TCP sockets. The server handles multiple HTTP methods, parses requests, manages file operations, and generates proper HTTP responses according to RFC specifications.

Built as part of the CSCI-P538 Computer Networks course, this implementation demonstrates fundamental concepts of network programming, including socket creation, request parsing, routing, and response generation.

VM Environment

Supported HTTP Methods

Method Description Endpoint
GET Retrieve files from the server Any valid file path
POST Submit data to be stored /post only
PUT Upload files to the server Any path
HEAD Get headers without body Any valid file path

Key Features

✓ Request Parsing: Properly parses HTTP request lines, headers, and bodies

✓ Path Resolution: Maps HTTP paths to filesystem locations securely

✓ MIME Type Detection: Automatically determines content types

✓ Error Handling: Gracefully handles 404, 405, and 500 errors

✓ Directory Traversal Protection: Prevents unauthorized file access

✓ Content-Length Support: Correctly handles request/response bodies

Network Architecture

The server runs on a virtualized two-node network using VirtualBox with Host-only networking. This setup allows for isolated testing while maintaining connectivity between the client and server VMs.

Server VM: Ubuntu 24.04 LTS (192.168.56.3:8080)

Client VM: Ubuntu 24.04 LTS (192.168.56.2)

Host Machine: 192.168.56.1
